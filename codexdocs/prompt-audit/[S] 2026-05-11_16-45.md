## 4. 시스템 프롬프트 수정 내역

문제: handoff 필드명이 final에서 변형됨

1. 어떤 행동이 문제였는가
   - coordinator final에서 `검증 필요 항목` 라벨을 쓰지 않고 `검증 gate`로 바꿨다.
   - 의미상 검증 계획은 있었지만, 협업 요구사항의 명시 필드명은 보존되지 않았다.

2. 기존 프롬프트의 어떤 문구가 원인이었는가
   - `Every delegate/report/final handoff must preserve five compact fields: 결정 사항, 미해결 문제, 리스크, 의존성, 검증 필요 항목.`
   - 이 문구는 필수 필드 목록을 제시하지만, 라벨명을 그대로 써야 한다고 명시하지 않았다.

3. 어떤 지시가 부족했는가
   - report/final에서 필드명을 바꾸지 말고 정확한 다섯 라벨을 verbatim으로 쓰라는 지시.

4. 어떤 문장으로 수정했는가
   - `Use the five handoff field labels verbatim: 결정 사항, 미해결 문제, 리스크, 의존성, 검증 필요 항목. Do not rename them, including in final answers.`

5. 수정 후 기대 행동은 무엇인가
   - 모든 delegate/report/final 메시지가 동일한 필드명을 사용한다.
   - 후속 멤버와 evaluator가 필드명을 안정적으로 인식한다.
