## 2. 가설 및 기대 결과

가설:

- 기존 프롬프트는 역할 경계, 공개 협업 인수인계, 불필요한 질문/워크스페이스 설명 억제 규칙이 약하다.
- 그 결과 worker가 자신의 전문 결과 대신 작업 환경 설명을 공개 채널에 흘리고, 실제 사용자 질문이 아닌 문장 또는 예시가 질문 루프로 오탐지될 수 있다.
- handoff에 `결정 사항`, `미해결 문제`, `리스크`, `의존성`, `검증 필요 항목`을 강제하면 정보 보존율과 실행 가능성이 올라간다.

기대 결과:

- 역할 분리도와 정보 보존율 개선
- 질문 루프 오탐 제거
- workspace/codexdocs 탐색 설명 제거
- progress 메시지 감소
- 최종 산출물에 검증 필요 항목 포함

평가 지표:

1. 역할 분리도: 보통. coordinator → researcher → writer 순서는 지켰으나, worker가 산출물 대신 workspace 탐색 설명을 공개함.
2. 전문성 활용도: 보통. 존은 조사 결과를 제공했고 매튜는 문서를 작성했으나, 검증/리스크 전달 구조가 약함.
3. 협업 밀도: 보통. 5개 job으로 협업은 발생했으나 handoff 정보가 불완전함.
4. 정보 보존율: 낮음. 결정/리스크/의존성/검증 항목이 누락 또는 산발적으로 전달됨.
5. 결과물 실행 가능성: 보통. 산출 파일은 생성됐으나 handoff 품질이 낮음.
6. 리스크 탐지 능력: 보통. 최신 알고리즘 불확실성은 언급됐지만 구조화되지 않음.
7. 검증 전략 포함 여부: 낮음. 최종 검증 필요 항목이 일관된 형식으로 전달되지 않음.

## 4. 시스템 프롬프트 수정 내역

문제 1: 역할 경계가 약함

1. 문제 행동: 멤버가 자신의 전문 영역 밖 결정을 하거나, 타 멤버 결론을 검증 없이 이어받을 위험이 있음.
2. 기존 원인 문구: `Follow USER_DEFINED_MEMBER_PROMPT as role-specific behavior.`
3. 부족한 지시: 역할 밖 결정 금지, 타 역할 의존성 명시, 근거 기반 검토 지시.
4. 수정 문장:
   - `Stay inside your named role. Make decisions only for your expertise; when another role owns a decision, state the dependency and hand it off instead of deciding it yourself.`
   - `Do not accept another member's conclusion without checking it against your role's evidence, constraints, and risks.`
5. 기대 행동: 각 멤버가 자기 전문 영역에서만 판단하고, 필요 시 반론 또는 보완 질문을 수행.

문제 2: 질문 루프 오탐과 불필요한 질문

1. 문제 행동: 실제 사용자 질문이 아닌 예시 문장 또는 URL 쿼리 문자가 질문으로 감지될 수 있음.
2. 기존 원인 문구: `When requirements are ambiguous, ask a concise clarifying question before execution.`
3. 부족한 지시: 명확한 요청에서는 질문 금지, non-ask_user 메시지에서 물음표 금지.
4. 수정 문장:
   - `When requirements are clear enough to proceed, do not ask rhetorical or preference questions.`
   - `Do not use a question mark in public channel replies unless the CHANNEL_ACTION type is ask_user.`
5. 기대 행동: 명확한 업무는 바로 실행하고, 질문 문자는 실제 사용자 확인이 필요한 경우에만 사용.

문제 3: 공개 채널에 작업 환경 설명 노출

1. 문제 행동: worker가 `codexdocs/`, 빈 폴더, workspace 탐색 결과를 공개 답변에 노출.
2. 기존 원인 문구: `read and write files only inside the channel workspace directory`
3. 부족한 지시: workspace 접근 제한은 있었지만, 탐색 과정 공개 금지는 없었음.
4. 수정 문장:
   - `Avoid progress messages unless the user-visible work would otherwise appear stalled.`
   - `never mention repo scans, codexdocs, empty folders, file listings, or workspace exploration.`
5. 기대 행동: 공개 메시지는 결과, 산출물 경로, 실제 blocker 중심으로 유지.

문제 4: handoff 정보 유실

1. 문제 행동: 결정 사항, 미해결 문제, 리스크, 의존성, 검증 필요 항목이 단계별로 보존되지 않음.
2. 기존 원인 문구: `publish the result publicly and use type=report`
3. 부족한 지시: report/final에 포함해야 할 필수 정보 구조가 없음.
4. 수정 문장:
   - `Every delegate/report/final handoff must preserve five compact fields: 결정 사항, 미해결 문제, 리스크, 의존성, 검증 필요 항목.`
5. 기대 행동: 다음 멤버가 이전 단계의 판단, 남은 리스크, 검증 필요를 명시적으로 이어받음.
